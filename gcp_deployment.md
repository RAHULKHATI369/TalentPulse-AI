# GCP Production Deployment Guide - TalentPulse AI

This guide walks you through deploying the TalentPulse AI application to **Google Cloud Platform (GCP)** under the Project ID `talentpulse-ai-501608` using **Google Cloud Run** and **Google Cloud SQL**.

---

## Prerequisites

Before deploying, ensure you have completed the following steps:

1. **Install Google Cloud SDK**: Install the [gcloud CLI](https://cloud.google.com/sdk/docs/install) on your local machine.
2. **Authenticate your account**:
   ```bash
   gcloud auth login
   gcloud auth application-default login
   ```
3. **Configure Project**:
   ```bash
   gcloud config set project talentpulse-ai-501608
   ```
4. **Billing Enabled**: Make sure billing is enabled for your project `talentpulse-ai-501608` in the GCP console.

---

## Step 1: Create a Production Cloud SQL Database

1. Go to the Cloud SQL Console and click **Create Instance**.
2. Select **MySQL** or **MariaDB** (MySQL 8.0 is recommended).
3. Set your Instance ID (e.g., `talentpulse-db`) and configuration details.
4. Under **Connections**, enable **Private IP** or **Public IP** depending on your network layout.
5. Create a database name: `talentpulse_db`
6. Create a user (e.g., `talentpulse_user`) and a secure password.
7. Copy the **Connection Name** for your instance. It looks like:
   `talentpulse-ai-501608:us-central1:talentpulse-db`

---

## Step 2: Secret Manager (Recommended for Gemini API Key)

To secure your Gemini API Key in production:
1. Go to **Secret Manager** in the GCP Console.
2. Create a secret named `GEMINI_API_KEY` and insert your API key as the secret value.
3. Grant the Cloud Run Service Account the **Secret Manager Secret Accessor** role (`roles/secretmanager.secretAccessor`).

---

## Step 3: Run the Automated Deployment Script

We have provided a deployment script [deploy.sh](file:///c:/Users/a/OneDrive/Desktop/TalentPulse%20AI/deploy.sh) in the project root.

To deploy, run the script from your terminal:
```bash
# On Linux/macOS or Git Bash/WSL on Windows
chmod +x deploy.sh
./deploy.sh
```

### Script Prompts:
1. **Enter GEMINI_API_KEY**: Paste your API key here (or leave empty to run in simulated mock AI mode).
2. **Enter Cloud SQL Connection Name**: Paste the connection name copied in Step 1 (e.g., `talentpulse-ai-501608:us-central1:talentpulse-db`).
3. **Enter Production DATABASE_URL**: Paste your MySQL SQLAlchemy connection string. Format:
   `mysql+pymysql://talentpulse_user:password@/talentpulse_db?unix_socket=/cloudsql/talentpulse-ai-501608:us-central1:talentpulse-db`

*Note: If all inputs are left blank, the application will fallback to using a local SQLite database (`talentpulse.db`) inside the Cloud Run container.*

---

## Step 4: Verification

Once the script finishes, it will print the **Service URL** generated by Google Cloud Run.
1. Open the Service URL in your browser.
2. Verify that the custom icon and styling load correctly.
3. Test uploading candidate resumes (PDF, TXT, or DOCX formats) to ensure the backend analyzes them correctly.
